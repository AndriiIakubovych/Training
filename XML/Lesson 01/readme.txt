Написать XML-документ для хранения данных о рецептах, т. е. книгу рецептов. Каждый рецепт (recipe) будет иметь следующую структуру, т. е. хранить следующие данные:
1) название (title) – название рецепта;
2) фото (photo) – необязательный элемент, который содержит фото блюда;
3) ингредиенты (ingredients): ингредиент (item) – название ингредиента, количество (quantity) – необходимое количество;
4) приготовление (preparation) – собственно сам рецепт;
5) порции (serving) – выход;
6) калорийность (energy) – необязательный элемент, который содержит информацию о калорийности блюда: белки (proteins), жиры (fats), углеводы (carbs);
Следует отметить, что ингредиентов при приготовлении может быть больше одного, а дочерние элементы в элементе energy (если он присутствует) могут размещаться в любом порядке. Кроме того, ряд элементов могут иметь дополнительные свойства, которые в XML-документе отображаются с помощью атрибутов:
a) родительский элемент каждого отдельного рецепта (recipe) обязательно будет иметь идентификатор;
b) элемент quantity содержит обязательно информацию о единицах измерения (граммы, литры, ст. ложки и т. д.);
c) элемент preparation может содержать данные (по желанию): степень сложности (может содержать одно из следующих значений: для начинающих (по умолчанию), для опытных, для мастеров), время приготовления, дополнительные характеристики с возможными значениями (для вегетарианцев, для диабетиков, низкокалорийное, можно замораживать).
